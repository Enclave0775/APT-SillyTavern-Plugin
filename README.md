# Auto Prompt Toggler (APT)

這是一個 SillyTavern 的擴充功能，能夠根據聊天訊息內容自動切換 Chat Completion 提示詞（Prompt）的啟用狀態。

## ✨ 功能特點

* **多語言介面**：支援 English、繁體中文、简体中文，可在插件設定區即時切換。
* **自動化控制**：設定 Regex 規則，當 AI 回覆或使用者輸入符合條件時，自動觸發提示詞狀態改變。
* **多條件觸發**：包含條件支援每行一個 Regex，並可選擇「任一條件符合 (OR)」或「所有條件皆符合 (AND)」。
* **排除條件**：可設定排除 Regex；只要排除詞出現，即使包含條件符合也不觸發。
* **三欄式拖曳分類介面**：
    * **未選擇的提示詞**：不受目前規則影響。
    * **觸發時啟用 / 結束時關閉**：符合條件時開啟，未符合時關閉。
    * **觸發時停用 / 結束時還原**：符合條件時關閉，未符合時開啟。
* **提示詞搜尋過濾**：在規則編輯器中快速搜尋大量提示詞。
* **規則名稱**：可替規則設定名稱，方便在列表中識別。
* **檢索層數 (Search Depth)**：每條規則可自訂要往前檢查幾則訊息；`1` 代表最新訊息，`0` 代表檢查全部歷史訊息。
* **偵測來源**：支援聊天顯示內容與原始訊息內容。
* **偵測對象**：支援 AI 輸出、使用者輸入或兩者。
* **全域 Profile 管理**：全域規則可建立多組 Profile，並支援切換、重新命名、匯入、匯出、清空。
* **Preset 專屬規則**：規則可綁定目前 API Preset，會跟隨 Preset 自動儲存與切換。
* **Global / Preset 雙向移動**：可將規則在全域與目前 Preset 之間移動。
* **匯入 / 匯出**：支援規則與設定檔匯入匯出，並保留舊格式相容。
* **通知開關**：可選擇是否在提示詞被開啟時顯示 Toast 通知。
* **受控提示詞搜尋**：可搜尋目前全域 Profile 與目前 Preset 中被規則控制的提示詞，並顯示控制來源、規則名稱與套用行為。
* **穩定訊息檢查**：以 SillyTavern 聊天資料為主要檢查來源，降低畫面渲染、虛擬化或只顯示尾端訊息造成的漏判機率。
* **規則衝突處理**：多條規則同時控制同一提示詞時，會依「觸發動作優先於未觸發反向動作」、「Preset 優先於 Global」、「列表中較後面的規則優先」決定最終狀態。
* **聊天補全主 API 保護**：僅在 SillyTavern 主 API 模式為「聊天補全」時執行切換；這不限制「聊天補全來源」供應商，可避免在文字補全、NovelAI、KoboldAI 等不相容提示詞結構下誤操作。
* **穩定性保護**：包含 HTML 轉義、匯入資料清洗、無效 Regex 防護、Regex 快取、Prompt Order 安全查找、Preset 規則遷移與相容處理。

## 🚀 使用方法

1. **安裝插件**
    1. 開啟 SillyTavern。
    2. 點擊頂部工具列中的「擴充功能 (Extensions)」。
    3. 點擊「安裝擴充功能 (Install Extension)」。
    4. 輸入：`https://github.com/Enclave0775/APT-SillyTavern-Plugin`
    5. 選擇安裝範圍並完成安裝。

2. **切換介面語言**
    * 開啟 Extensions 面板中的 **Auto Prompt Toggler Rules / 自動提示詞切換規則**。
    * 在 **介面語言 / Interface Language** 下拉選單選擇：
        * `繁體中文`
        * `简体中文`
        * `English`

3. **設定規則**
    * 選擇要編輯的全域 Profile，或使用目前 Preset 規則。
    * 點擊新增規則。
    * 設定規則名稱、偵測來源、偵測對象、觸發模式、Regex 包含/排除條件與檢查層數。
    * 將提示詞拖曳至：
        * **觸發時啟用 / 結束時關閉**
        * **觸發時停用 / 結束時還原**
    * 點擊儲存。

4. **實際運作**
    * 當最新訊息或指定檢索範圍內的訊息符合 Regex 條件時，插件會依規則強制切換提示詞狀態。
    * 若觸發模式為 **任一條件符合 (OR)**：包含條件中任一行 Regex 命中即可觸發。
    * 若觸發模式為 **所有條件皆符合 (AND)**：包含條件中每一行 Regex 都必須在同一則被檢查的訊息中命中才會觸發，例如：
      ```text
      出鞘
      劍
      ```
      代表同一則訊息需同時包含「出鞘」與「劍」。若要三個條件，就再新增第三行。
    * 排除條件為「任一排除 Regex 命中就不觸發」，例如包含條件填 `劍`、排除條件填 `木劍`，則有「劍」但不含「木劍」時才會觸發。
    * 未觸發時會套用相反狀態，而不是記住每個提示詞先前的手動狀態。
    * 若多條規則同時控制同一提示詞，最終狀態會由規則優先權決定，而不是依執行途中先後反覆覆蓋。

5. **搜尋受規則控制的提示詞**
    * 在插件設定面板中，可使用「搜尋受規則控制的提示詞」快速查找目前被規則控制的提示詞。
    * 搜尋範圍包含目前全域 Profile 與目前 Preset 的規則。
    * 可用提示詞名稱、提示詞 ID 或規則名稱搜尋。
    * 結果會顯示控制來源、規則名稱、觸發時啟用/停用行為，以及該規則目前是否啟用。

## 📂 匯入與匯出

* **全域 Profile 匯出 / 匯入**：針對目前 Profile 的規則備份與還原。
* **Preset 規則匯出 / 匯入**：針對目前 API Preset 綁定規則。
* **單一規則匯出**：可從規則列表匯出單條規則。
* **Preset 匯入防呆**：匯入外部 Preset 時，如果包含 APT 規則，會詢問是否保留。

## ⚠️ 注意事項

* 此插件依賴 SillyTavern 的 `promptManager` 與 Chat Completion Prompt 管理功能。
* 若目前主 API 模式不是「聊天補全」，APT 會避免執行提示詞切換與規則編輯，以免操作到不支援的提示詞結構；「聊天補全來源」選擇 OpenAI、自定義相容 OpenAI、OpenRouter、Google AI Studio 等不受此限制。
* 包含條件與排除條件皆使用 JavaScript `RegExp`，並以不分大小寫方式匹配；每行代表一條獨立 Regex。
* 無效 Regex 會被略過並顯示錯誤提示，不會中斷整個插件流程。
* 「觸發時停用 / 結束時還原」的還原邏輯是未觸發時強制開啟，不是回復到使用者手動切換前的狀態。
* 匯出檔名會自動過濾 Windows 不合法字元，避免 Profile 或 Preset 名稱造成下載檔名異常。
